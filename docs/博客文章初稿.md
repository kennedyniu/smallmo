## 一、环境准备

### 1.1 安装 Node.js

Node.js 是运行 Claude Code 的基础环境。

**下载方式：**
- 官网下载：https://nodejs.org/zh-cn/
- 推荐 LTS（长期支持）版本

**安装验证：**
打开终端（Windows 下按 Win+R，输入 cmd），输入：
```bash
node -v
npm -v
```
如果显示版本号，说明安装成功。

### 1.2 安装 Claude Code

Claude Code 是 Anthropic 推出的 AI 编程助手，通过百度千帆大模型驱动，我们可以在国内流畅使用。

**Step 1: 安装 Claude Code**

```bash
npm install -g @anthropic-ai/claude-code
```

**Step 2: 配置百度千帆大模型**

安装完成后，需要配置百度千帆作为模型驱动。在项目根目录（或者你希望开发的项目目录）创建 `.claude.json` 配置文件：

```json
{
    "model": "qianfan-code-latest",
    "env": {
        "CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC": 1,
        "API_TIMEOUT_MS": 600000,
        "ANTHROPIC_MODEL": "qianfan-code-latest",
        "ANTHROPIC_SMALL_FAST_MODEL": "qianfan-code-latest",
        "ANTHROPIC_DEFAULT_HAIKU_MODEL": "qianfan-code-latest",
        "ANTHROPIC_DEFAULT_SONNET_MODEL": "qianfan-code-latest",
        "ANTHROPIC_DEFAULT_OPUS_MODEL": "qianfan-code-latest"
    },
    "permissions": {
        "allow": [],
        "deny": []
    }
}
```

> **配置说明：**
> - `model` 和 `ANTHROPIC_MODEL` 等都设置为 `qianfan-code-latest`，这是百度千帆提供的代码模型
> - `API_TIMEOUT_MS` 设置为 600000（10分钟），避免大模型调用超时
> - `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC` 设为 1，减少非必要网络请求

**Step 3: 跳过官方账号验证**

首次安装完成后，需要跳过官方账号验证。在终端运行以下命令：

Windows:
```bash
echo '{"hasCompletedOnboarding": true}' > %USERPROFILE%\.claude.json
```

macOS / Linux:
```bash
echo '{"hasCompletedOnboarding": true}' > ~/.claude.json
```

**Step 4: 启动 Claude Code**

进入你的项目目录，启动 Claude Code：

```bash
cd 你的项目目录
claude
```

启动后，选择「信任此文件夹 (Trust This Folder)」，让 Claude Code 可以访问该项目中的文件。

### 1.3 配置百度千帆 API Key

要在 Claude Code 中使用百度千帆大模型，还需要获取百度千帆的 API 凭证。

**Step 1: 获取 API Key**

1. 访问百度千帆大模型平台：https://cloud.baidu.com/doc/qianfan/s/0mn2mnemj
2. 注册百度账号并完成实名认证
3. 登录百度智能云控制台，进入「千帆大模型平台」
4. 创建应用，填写应用名称（如「Claude Code 开发助手」）
5. 获取 **API Key** 和 **Secret Key**

**Step 2: 获取 Access Token**

百度千帆使用 OAuth2.0 获取访问令牌，需要通过 API Key 和 Secret Key 换取 token：

```bash
# 将 YOUR_API_KEY 和 YOUR_SECRET_KEY 替换为你获取的实际值
curl -X POST "https://aip.baidubce.com/oauth/2.0/token?grant_type=client_credentials&client_id=YOUR_API_KEY&client_secret=YOUR_SECRET_KEY"
```

返回的 JSON 中会包含 `access_token` 字段，复制这个值。

**Step 3: 配置 Access Token**

将获取到的 access_token 配置到环境变量中：

Windows (PowerShell):
```bash
$env:ANTHROPIC_API_KEY="你的access_token"
```

macOS / Linux:
```bash
export ANTHROPIC_API_KEY="你的access_token"
```

> **提示**：如果觉得每次启动都要配置比较麻烦，可以将环境变量写入系统的环境变量设置中（Windows 系统属性 → 环境变量），或者写入 `.bashrc` / `.zshrc` 文件。

> **注意**：百度千帆 Code-Plus 包月套餐费用为 **40元/月**，访问 https://console.bce.baidu.com/qianfan/resource/subscribe 购买。登录百度账号后，在「资源订阅」页面找到 Coding Plan 包月套餐进行购买即可。

### 1.4 安装微信开发者工具

微信小程序开发必须使用官方开发工具。

1. 下载地址：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
2. 安装并登录你的微信账号
3. 记得开启「开发辅助」-「调试基础库」

## 二、开始开发：把需求"说"出来

### 2.1 第一步：描述你的需求

我是这样告诉 Claude Code 的：

> 我想做一个记账小程序，名叫「记账」。主要功能包括：
> - 记收入和支出
> - 按月份查看账单
> - 图表展示收支占比
> - 支持多个账户
> - 可以导入导出数据
> - 界面要好看，最好有主题切换

然后 Claude Code 就开始工作了——它会分析需求、设计架构、写出代码。

### 2.2 沟通需求和方案

Claude Code 不会一次性写完所有代码，而是会分步骤跟我确认：

1. **确认功能模块** — 先列出要做哪些页面
2. **确认数据模型** — 讨论账单、账户、分类的数据结构
3. **确认 UI 设计** — 讨论颜色、布局、交互
4. **代码实现** — 每完成一个模块让我验收

我只需要用自然语言回复"可以"、"再改一下"、"这里不对"，就像和产品经理沟通一样。

### 2.3 迭代开发过程

举几个实际沟通的例子：

**场景 1：我要改主题颜色**
> 我：能不能把界面改成粉色的？我觉得更适合女生用。
> Claude：好的，我帮你增加了一个「可爱」主题，包含了粉色配色和圆润的字体。

**场景 2：发现一个 bug**
> 我：导出数据后打开是乱的，格式不对。
> Claude：明白了，是编码问题。我来修复 CSV 的中文编码。

**场景 3：新增功能**
> 我：我想加一个月度报表功能，能看到每月花了多少钱。
> Claude：收到，我帮你加一个统计页面，支持月度趋势图。

整个过程，我没有看过一行代码，全部是文字沟通。

## 四、认证、备案与上线

小程序开发完成后，还需要几步才能正式上线。

### 4.1 微信认证

1. 登录微信公众平台：https://mp.weixin.qq.com/
2. 注册小程序账号
3. 完成微信认证（个人类型无需认证，企业/组织需要认证）

### 4.2 小程序备案

根据国家相关规定，微信小程序需要进行备案：

1. 在公众平台提交备案信息
2. 填写小程序基本信息、服务类目
3. 个人小程序：填写个人身份信息即可
4. 等待审核（一般 7 个工作日）

### 4.3 提交审核

1. 在微信开发者工具上传代码
2. 在公众平台提交审核
3. 填写小程序信息：名称、描述、图标
4. 审核周期：通常 1-7 天

### 4.4 发布上线

审核通过后，在公众平台点击「发布」即可。

## 五、总结与心得

### 5.1 收获

- **真香警告**：原本以为编程很难，结果发现"说话"就能做软件
- **效率惊人**：传统开发可能需要几周，我只用了几天（每天沟通 1-2 小时）
- **成就感满满**：自己的小程序真的上线了，还能分享给朋友用

### 5.2 经验分享

1. **需求要描述清楚** — 越详细的需求，AI 理解得越准确
2. **及时验收** — 每完成一个功能就测试，避免积重难返
3. **善用截图** — 发现问题直接截图发给 AI，它能更快理解
4. **保持耐心** — AI 不是万能的，有时候需要多轮沟通才能达到预期

### 5.3 未来展望

这个小程序目前还在持续迭代中，我准备后续添加：
- 预算管理功能
- 账单提醒
- 多设备同步
- 账单标签

如果你也想做自己的小程序，不妨试试这个方法——也许下一个上线的就是你的创意！
